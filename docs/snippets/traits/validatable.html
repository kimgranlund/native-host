Validatable

Composable trait for declarative form validation. Attach rules to any element, call validate(), and react to ui-valid / ui-invalid events. Sets [invalid] and aria-invalid attributes automatically.

Required Field

Uses ValidateController directly on a wrapper div with a ui-input inside.

Validate 
import { ValidateController } from '@nonoun/native-ui';

const field = document.getElementById('field');
const validator = new ValidateController(field);
validator.rules = [
  ValidateController.required('Name is required'),
];

field.addEventListener('ui-invalid', (e) => {
  errorEl.textContent = e.detail.message;
});
field.addEventListener('ui-valid', () => {
  errorEl.textContent = '';
});

// Validate on button press
button.addEventListener('ui-press', () => {
  validator.validate();
});

Multiple Rules (min length + pattern)

Validate 
validator.rules = [
  ValidateController.required('Password is required'),
  ValidateController.minLength(8, 'Must be at least 8 characters'),
  ValidateController.pattern(/[A-Z]/, 'Must contain an uppercase letter'),
  ValidateController.pattern(/[0-9]/, 'Must contain a number'),
];

Validate on Blur

Focus the input, type something, then tab away. Validation runs on blur. 

const emailInput = field.querySelector('ui-input');
emailInput.addEventListener('focusout', () => {
  validator.validate();
});

API

Methods

MethodReturnsDescription
validate(value?) boolean Runs all rules against the value (or auto-detects from child input). Sets [invalid] and dispatches events.
clearValidation() — Removes [invalid] and clears error message.

Static Rule Helpers

HelperDescription
ValidateController.required(msg) Fails if value is empty or whitespace-only.
ValidateController.minLength(n, msg) Fails if value length is less than n.
ValidateController.pattern(regex, msg) Fails if value does not match the regex.

Events

EventDetailDescription
ui-invalid { message } Fired when validation fails. message is the first failing rule's message.
ui-valid — Fired when validation passes after a previous failure.

CSS

SelectorDescription
[invalid] Set on the host element when validation fails. Use to style error state on child inputs.
