Customization Patterns

State-conditional intent

Override --_* locals in a state selector to change intent on hover, focus, etc. 

/* Neutral button → accent on hover */
ui-button.cta:hover {
  --_ink:        var(--accent-ink);
  --_ink-strong: var(--accent-ink-strong);
  --_surface:    var(--accent-surface);
  --_panel:      var(--accent-panel);
  --_border:     var(--accent-border);
}

/* Danger intent on focus (e.g. delete confirmation) */
ui-button.danger-focus:focus-visible {
  --_ink:        var(--danger-ink);
  --_surface:    var(--danger-surface);
}

Debug state attributes (force-*)

Every pseudo-state with a visual effect has a force-* attribute mirror for DevTools debugging and state demos. Add the attribute to the host element to simulate the pseudo-state.

Pseudo Attribute Components :hover [force-hover] All interactive components + ui-card[interactive] :active [force-active] ui-button, ui-option, ui-command-item, ui-tab, ui-segment :focus [force-focus] ui-input, ui-textarea, ui-input-otp :focus-visible [force-focus-visible] All components with focus rings 
<!-- Toggle in DevTools to simulate hover -->
<ui-button force-hover>Hovered</ui-button>

<!-- Compound states work too -->
<ui-checkbox force-hover aria-checked="true">Checked + Hovered</ui-checkbox>

<!-- Internal children: force-* on host triggers all children -->
<ui-calendar force-hover>...</ui-calendar>

Scoped intent override

Override locals on a container to affect all children. 

/* All controls inside .danger-zone get danger colors */
.danger-zone {
  --_ink:           var(--danger-ink);
  --_ink-strong:    var(--danger-ink-strong);
  --_surface:       var(--danger-surface);
  --_border:        var(--danger-border);
  --_border-muted:  var(--danger-border-muted);
}
